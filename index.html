Introduction

Embedded Store Engine API

Drop a fully working storefront into any site or app. This reference covers everything you need to fetch store data, list products, and create orders.

Used in every example on this page — swap it for testing.

Every request is scoped to a store. Point the SDK or your own HTTP client at vm.embedded.store, pass your store ID, and you're ready to render products and take orders.

Quickstart

Three calls get a working checkout flow: fetch the store, list products, then create an order when a customer checks out.

  1. Fetch your store details to confirm the ID is live.
  2. List products to render your catalog.
  3. Create an order once the customer submits their cart.

Authentication

Every request needs an API key in the Authorization header. Keys are scoped per store and available from your dashboard.

HeaderValue
AuthorizationBearer <your_api_key>
GET /v1/stores/{store_id}

Get store info

Returns the public details of a store — name, currency, and status. Use this to confirm a store ID before rendering anything.

Path parameters

ParamTypeDescription
store_idstringrequiredThe store's unique identifier.
GET /v1/stores/{store_id}/products

List products

Returns all active products for a store, ready to render into a grid or list.

Query parameters

ParamTypeDescription
limitintegeroptionalMax results per page. Defaults to 20.
cursorstringoptionalPagination cursor from a previous response.
POST /v1/stores/{store_id}/orders

Create an order

Creates a new order from a cart of line items. Returns an order object with its current state.

Body parameters

ParamTypeDescription
itemsarrayrequiredList of { product_id, quantity } objects.
customer_emailstringoptionalUsed for order confirmation and receipts.
PUT /v1/stores/{store_id}/orders/{order_id}

Update an order

Transitions an order to a new state — for example, marking it as paid once payment is confirmed.

ParamTypeDescription
statusstringrequiredOne of the valid order states. See order states below.

Errors

Errors return a standard shape with an HTTP status code, a short error code, and a human-readable message.

StatusCodeMeaning
401unauthorizedMissing or invalid API key.
404not_foundStore, product, or order doesn't exist.
422validation_errorOne or more fields failed validation.

Order states

Orders move through a fixed set of states as they're placed, paid, fulfilled, and closed. The full set spans 29 states across payment, fulfillment, and post-sale flows — pending, paid, packed, shipped, delivered, refunded, and more.

Public API Endpoints

 Available Endpoints

Read Data

GET?state=products— List products
GET?state=product&id={id}— Single product
GET?state=categories— List categories
GET?state=search&q={query}— Search products
GET?state=discounts— Active discounts
GET?state=site— Store info
GET?state=orders&email={email}— Order history
GET?state=cart&cart_id={id}— Cart contents

Cart & Checkout

POST?state=cart_create— Create cart
POST?state=cart_add— Add to cart
POST?state=cart_update— Update quantity
POST?state=cart_remove— Remove item
POST?state=checkout_create— Create checkout
POST?state=checkout_complete— Complete checkout
POST?state=order— Place order

Auth via X-API-Key header, Authorization: Bearer {key}, or ?api_key={key}

JAVASCRIPT SDK

Drop-in storefront SDK for external websites

Include Script
<script src="http://localhost:8016/store-access/35/sdk/vm-store.js"></script>

// Initialize the SDK

const store = new VMStore({
  storeId: '35',
  apiKey: 'YOUR_API_KEY'
});

store.ui.injectStyles();
store.ui.productGrid('#shop');
store.ui.cartBadge('#cart-icon');
store.ui.cartDrawer('#cart');

 SDK Methods

Products.products.list() / .get(id) / .search(q) / .byCategory(id)
Cart.cart.add(id, qty) / .update(id, qty) / .remove(id) / .get() / .clear()
Checkout.checkout.redirect({returnUrl}) / .create() / .complete()
UI.ui.productGrid(el) / .cartBadge(el) / .cartDrawer(el) / .injectStyles()
Events.on('cart:updated' | 'cart:item-added' | 'checkout:completed', cb)

// Raw fetch example

fetch('http://localhost:8016/store-access/35/?state=products', {
  headers: { 'X-API-Key': 'YOUR_API_KEY' }
})
.then(res => res.json())
.then(data => console.log(data));
